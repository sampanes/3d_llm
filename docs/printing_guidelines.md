AI-generated 3D models require extra scrutiny before printing. This guide covers how to evaluate, prepare, and print models generated by the 3D LLM pipeline.

## Before You Slice: Evaluating Printability

### Visual Inspection in OpenSCAD

Before exporting to STL, preview the model in OpenSCAD:

1. **F5 (Preview)** — Quick render. Look for:
   - Red/pink surfaces = flipped normals or non-manifold edges
   - Missing faces = incomplete boolean operations
   - Floating geometry = unconnected parts

2. **F6 (Full Render)** — CGAL render. This is what the STL export will produce. If F6 fails or shows errors in the console, the model needs fixing.

### Automated Checks

```bash
# Run the built-in printability check (if available)
python main.py check output/stl/my_model.stl

# Or use admesh for quick analysis
admesh output/stl/my_model.stl
```

### What to Look For

| Check                | What It Means                                           | Action If Failed          |
|----------------------|---------------------------------------------------------|---------------------------|
| Watertight mesh      | No holes in the surface                                 | Repair in Meshmixer       |
| Manifold edges       | Every edge shared by exactly 2 faces                    | Re-generate or repair     |
| Positive volume      | Model has inside and outside                            | Check normals direction   |
| Min. wall thickness  | Walls ≥ 0.8mm for 0.4mm nozzle                         | Increase in parameters    |
| No self-intersection | Geometry doesn't pass through itself                    | Simplify boolean ops      |
| Reasonable size       | Model fits on your print bed                            | Scale or adjust parameters|

## Recommended Slicer Settings

### General Defaults (PrusaSlicer / Cura)

These settings work well as starting points for AI-generated models:

| Setting              | Value             | Notes                                     |
|----------------------|-------------------|-------------------------------------------|
| Layer Height         | 0.2 mm            | Good balance of speed and quality          |
| First Layer Height   | 0.25 mm           | Better bed adhesion                        |
| Perimeters           | 3                 | Extra strength for potentially thin walls  |
| Top/Bottom Layers    | 4                 | Good surface quality                       |
| Infill               | 20%               | Adjust based on functional needs           |
| Infill Pattern       | Gyroid             | Strong in all directions                   |
| Supports             | On (tree)          | AI models often have unexpected overhangs  |
| Support Threshold    | 45°               | Standard for FDM                           |
| Brim                 | 5mm                | Recommended for first prints of new models |
| Speed                | 50 mm/s            | Conservative; increase after test print    |

### For Functional / Mechanical Parts

| Setting              | Value             | Why                                        |
|----------------------|-------------------|-------------------------------------------|
| Perimeters           | 4–5               | Maximum wall strength                      |
| Infill               | 40–60%            | Load-bearing capability                    |
| Layer Height         | 0.16 mm           | Better layer adhesion                      |
| Material             | PETG / ABS        | Better mechanical properties than PLA      |

### For Quick Test Prints

| Setting              | Value             | Why                                        |
|----------------------|-------------------|-------------------------------------------|
| Layer Height         | 0.28–0.32 mm      | Fast layer times                           |
| Perimeters           | 2                 | Minimum viable walls                       |
| Infill               | 10%               | Just enough internal structure             |
| Supports             | Off (if possible)  | Faster print, easier cleanup              |
| Speed                | 80–100 mm/s       | Prioritize speed over quality              |

## Common Issues with AI-Generated Geometry

### 1. Non-Manifold Edges

**What it looks like:** Slicer shows warnings, sliced preview has gaps or missing walls.

**Why it happens:** LLMs sometimes create geometry where edges are shared by more than 2 faces, or faces that touch only at an edge (T-junction geometry).

**Fix:**
- In OpenSCAD: ensure all `difference()` cuts fully penetrate (extend cutters by 0.01mm beyond the surface)
- Use mesh repair: Meshmixer → Edit → Inspector → Auto Repair All

### 2. Zero-Thickness Walls

**What it looks like:** Walls thinner than your nozzle diameter disappear in the slicer.

**Why it happens:** LLMs sometimes calculate wall thickness incorrectly, especially when subtracting shells.

**Fix:**
- Check that `wall_thickness` ≥ 2× nozzle diameter
- In slicer: enable "Detect thin walls" (PrusaSlicer) or "Print thin walls" (Cura)

### 3. Internal Geometry / Trapped Volumes

**What it looks like:** Slicer shows infill in unexpected places, or the model has hidden internal structures.

**Why it happens:** Boolean `union()` operations sometimes leave internal faces where two solids overlap.

**Fix:**
- In OpenSCAD: the CGAL render (F6) usually cleans these up
- If persists: use `convexity` parameter or simplify the geometry

### 4. Overhangs Beyond 45°

**What it looks like:** Droopy, stringy material on unsupported faces.

**Why it happens:** LLMs often don't consider print orientation when designing geometry.

**Fix:**
- Add supports in slicer (tree supports minimize material)
- Consider splitting the model and printing in parts
- Re-orient the model — many overhangs can be eliminated by rotating 90°

### 5. Disconnected / Floating Parts

**What it looks like:** Parts of the model are floating in mid-air, not connected to anything.

**Why it happens:** LLMs sometimes position features with `translate()` but don't connect them to the main body.

**Fix:**
- Add a bridge between floating parts
- Use `hull()` to connect separate volumes
- Move the feature to touch the main body

### 6. Incorrect Scale

**What it looks like:** Model is 1000× too large or too small.

**Why it happens:** Confusion between mm and inches, or unitless dimensions.

**Fix:**
- Always verify dimensions in the slicer's 3D viewport
- OpenSCAD works in mm by default; CadQuery defaults to mm

### 7. Very High Polygon Count

**What it looks like:** STL file is huge (>50MB), slicer is slow.

**Why it happens:** `$fn = 200` or higher on many curved surfaces, or `minkowski()` on complex shapes.

**Fix:**
- Reduce `$fn` to 60 for most applications, 120 for threads
- Use `$fa` and `$fs` instead of `$fn` for adaptive resolution

## Material Recommendations

| Part Type                      | Recommended Material | Why                                          |
|--------------------------------|---------------------|----------------------------------------------|
| Prototypes / test prints       | PLA                 | Cheap, easy to print, good detail            |
| Functional parts (indoor)      | PETG                | Stronger than PLA, slight flex               |
| Functional parts (outdoor)     | ASA or ABS          | UV resistant, heat resistant                 |
| Snap-fit parts                 | PETG or Nylon       | Flexibility for snap features                |
| Threaded parts                 | PETG                | Good layer adhesion, threads less likely to strip |
| Cable clips / clamps           | TPU (95A)           | Flexible, won't break cables                 |
| Enclosures / boxes             | PLA or PETG         | Good dimensional accuracy                    |
| Moving parts / hinges          | PETG or Nylon       | Wear resistance, fatigue resistance           |

> [!TIP]
> For first prints of AI-generated models, always use PLA. It's the most forgiving material and makes it easy to spot geometry issues. Switch to functional materials once the design is validated.

## Post-Processing Tips

### Removing Supports
- Let the print cool completely before removing supports
- Use flush cutters for clean removal
- Sand support contact points with 150-grit, then 220-grit

### Improving Surface Quality
- **Sanding:** Progressive grits — 150 → 220 → 400 → 800
- **Filler primer:** Spray-on primer fills layer lines (Rust-Oleum Filler Primer works well)
- **Epoxy coating:** XTC-3D or similar for a smooth, glossy finish
- **Vapor smoothing:** ABS only — acetone vapor smooths surfaces (do in ventilated area)

### Improving Strength
- **Annealing PLA:** Bake at 60–70°C for 1 hour → increases crystallinity and heat resistance
- **Annealing PETG:** Bake at 75–85°C for 1 hour
- **Epoxy infusion:** Coat with thin epoxy for waterproofing and strength

### Fixing Fit Issues
- **Too tight:** Sand the mating surfaces or reprint with +0.1mm tolerance
- **Too loose:** Apply a thin layer of CA glue (superglue) to build up the surface
- **Threads don't work:** Chase with a real tap/die, or reprint with +0.2mm thread tolerance

### Assembly
- **PLA to PLA:** CA glue (superglue) or epoxy
- **PETG to PETG:** CA glue works; epoxy is stronger
- **Threaded inserts:** Use a soldering iron to heat-set M3/M4 brass inserts for strong threads

## Quick Checklist

Use this checklist before every print of an AI-generated model:

- [ ] Model previews correctly in OpenSCAD (F5) or CQ-editor
- [ ] Full render (F6) completes without errors
- [ ] STL file size is reasonable (< 20MB for most parts)
- [ ] Imported into slicer without warnings
- [ ] Dimensions match expectations (measure in slicer)
- [ ] No red/error highlights in sliced preview
- [ ] Wall thickness ≥ 0.8mm everywhere
- [ ] Overhangs have supports (or are < 45°)
- [ ] First layer preview looks good (adequate bed contact)
- [ ] Estimated print time is reasonable
- [ ] Using appropriate material for the part's function
- [ ] Brim enabled for first print of this design

> [!WARNING]
> AI-generated models have a higher-than-average rate of subtle geometry errors. Always do a careful visual inspection of the sliced preview (layer-by-layer) before starting a long print. A 5-minute inspection can save hours of wasted filament.
